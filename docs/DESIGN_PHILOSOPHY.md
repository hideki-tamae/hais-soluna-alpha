# 文明OS (Satsuma-v1.1) 設計思想：弾性の黄金基準

## 1. 正常性（Normalcy）の再定義
本システムは、ユーザーを「平均的な正常」に矯正することを目的としない。
むしろ、迷走神経の低下（背側迷走神経状態）を「エネルギーの充填」と捉え、そこからの回復（Sync）の勾配を「共鳴資産（Resonance Index）」として価値化する。

## 2. 数理モデルの透明性（logicVersion: Satsuma-v1.1）
- **DEGRADATION_THRESHOLD (0.7)**: 腹側迷走神経のブレーキが外れる生理学的変異点。
- **Elasticity Volume (弾性総量)**: 回復の軌跡の積分。
- **Action Multiplier**: ケア行動 A に対し、 1 + (A / 100) を乗算。知（生体）と行（貢献）の合一。

## 3. 不言実行（Silent Intervention）の原則
UIは説教を行わない。SyncProposal は、ユーザーが「本来の自分」を思い出すための、静かな波紋（Ripple）としてのみ機能する。
